# ChatBridge Apple 平台类微信聊天项目需求文档

> 项目名暂定：**ChatBridge**  
> 项目类型：Apple 平台类微信聊天 / IM App  
> 支持平台：iOS 优先，后续扩展 iPadOS、macOS、visionOS、watchOS

---

## 1. 项目概述

### 1.1 项目名称

```text
ChatBridge
```

### 1.2 项目定位

ChatBridge 是一款面向 Apple 平台的即时通讯 App，目标是实现类似微信的核心聊天体验，支持：

- 单聊
- 群聊
- 会话列表
- 消息收发
- 图片、语音、视频、文件消息
- 消息撤回
- 消息删除
- 消息已读 / 未读
- 会话置顶
- 会话免打扰
- 本地搜索
- 多端同步
- 本地数据库加密
- 媒体文件缓存
- 离线消息同步

### 1.3 支持平台

第一阶段优先支持：

```text
iOS
```

后续扩展：

```text
iPadOS
macOS
visionOS
watchOS
```

### 1.4 技术方向

Apple 平台建议采用：

```text
Swift
UIKit / SwiftUI
Combine
Swift Concurrency
SQLite / WCDB / GRDB
Keychain
UserNotifications
PushKit 可选
AVFoundation
PhotosUI
FileProvider 可选
```

## 2. 产品目标

### 2.1 核心目标

实现一个完整的移动端 IM 聊天系统，包括：

1. 用户登录后能看到会话列表。
2. 用户可以进入单聊或群聊。
3. 用户可以发送文本、图片、语音、视频、文件等消息。
4. 消息支持本地持久化。
5. 会话列表支持未读数、最后一条消息、时间排序。
6. 消息支持撤回、删除、重发。
7. 本地支持聊天记录搜索。
8. 弱网、断网、重连情况下消息状态可恢复。
9. 支持数据库迁移、加密和修复能力。
10. 为多端同步预留结构。

### 2.2 非目标范围

第一阶段暂不实现：

- 朋友圈
- 视频号
- 支付
- 小程序
- 公众号完整体系
- 企业通讯录
- 微信红包
- 音视频通话
- 直播间
- 游戏中心

可以预留扩展入口，但不作为第一阶段核心需求。

## 5. 账号模块需求

### 5.1 登录

#### 功能说明

用户通过手机号、账号密码、验证码或第三方账号登录。

#### 需求点

- 支持账号登录。
- 支持登录态缓存。
- 支持 token 自动刷新。
- 支持登录过期后重新登录。
- 支持退出登录。
- 支持多账号数据隔离。

#### 本地数据隔离

每个账号独立目录：

```text
account_xxx/
  main.db
  search.db
  file_index.db
  media/
  cache/
```

#### 验收标准

- 登录成功后进入会话列表。
- 退出登录后不再展示原账号数据。
- 切换账号后不会串数据。
- 删除账号数据时只清理当前账号目录。

## 6. 联系人模块需求

### 6.1 联系人列表

#### 功能说明

展示当前用户的联系人列表。

#### 需求点

- 联系人按首字母排序。
- 支持联系人搜索。
- 支持联系人头像、昵称、备注展示。
- 支持星标联系人。
- 支持拉黑联系人。
- 支持删除联系人。
- 支持陌生人临时会话。

#### 字段能力

联系人应包含：

```text
contact_id
user_id
nickname
remark
avatar_url
type
is_starred
is_blocked
is_deleted
source
updated_at
created_at
```

#### 验收标准

- 联系人列表加载速度稳定。
- 联系人备注优先于昵称展示。
- 被拉黑用户不能正常发起会话。
- 删除联系人后本地联系人列表不再展示。

## 7. 会话模块需求

### 7.1 会话列表

#### 功能说明

展示所有单聊、群聊、系统会话。

#### 展示内容

每个会话展示：

- 头像
- 标题
- 最后一条消息摘要
- 最后一条消息时间
- 未读数
- 是否置顶
- 是否免打扰
- 草稿
- 发送失败状态

#### 排序规则

优先级：

```text
置顶会话
按 sort_ts 倒序
普通会话
按 sort_ts 倒序
```

#### 会话表冗余字段

为了提高会话列表性能，需要在 `conversation` 表冗余：

```text
last_message_id
last_message_time
last_message_digest
unread_count
sort_ts
draft_text
is_pinned
is_muted
```

#### 验收标准

- 新消息到达后，会话列表自动更新。
- 最新会话移动到顶部。
- 置顶会话始终排在普通会话前面。
- 草稿优先展示。
- 未读数展示准确。
- 大量会话下滚动不卡顿。

### 7.2 会话置顶

#### 功能说明

用户可以将重要会话置顶。

#### 需求点

- 支持单聊置顶。
- 支持群聊置顶。
- 支持取消置顶。
- 置顶状态需要本地持久化。
- 置顶状态支持多端同步。

#### 验收标准

- 置顶后会话立即移动到顶部区域。
- 取消置顶后回到正常排序。
- 重启 App 后置顶状态仍然存在。

### 7.3 会话免打扰

#### 功能说明

用户可以设置某个会话免打扰。

#### 需求点

- 免打扰会话收到消息不弹通知。
- 会话列表仍然显示未读数。
- 可以取消免打扰。
- 群聊和单聊均支持。

#### 验收标准

- 免打扰开启后，收到消息不响铃、不震动。
- 未读数仍能正常累加。
- 关闭免打扰后通知恢复。

### 7.4 草稿

#### 功能说明

用户输入但未发送的内容需要保存为草稿。

#### 需求点

- 离开聊天页时保存草稿。
- 回到聊天页时恢复草稿。
- 会话列表展示草稿。
- 发送成功后清空草稿。
- 草稿更新时间影响会话排序，可根据产品策略决定。

#### 验收标准

- App 重启后草稿仍然存在。
- 发送消息后草稿清空。
- 删除会话后草稿清空。

## 8. 消息模块需求

### 8.1 消息类型

第一阶段支持：

| 类型 | 说明 |
|---|---|
| 文本消息 | 普通文本、表情、@ |
| 图片消息 | 原图、缩略图 |
| 语音消息 | 录音、播放、未读红点 |
| 视频消息 | 视频缩略图、播放 |
| 文件消息 | 文件名、大小、下载状态 |
| 系统消息 | 入群、退群、撤回提示 |
| 表情消息 | emoji、自定义表情 |
| 引用消息 | 回复某条消息 |
| 撤回消息 | 撤回后展示提示 |

### 8.2 文本消息

#### 功能说明

用户可以发送普通文本消息。

#### 需求点

- 支持纯文本。
- 支持 emoji。
- 支持 URL 识别。
- 支持手机号识别，可选。
- 支持 @ 用户，群聊场景。
- 支持长文本折叠，可选。
- 支持复制。
- 支持转发。
- 支持删除。
- 支持撤回。

#### 发送流程

```text
用户输入文本
↓
点击发送
↓
生成 client_msg_id
↓
本地插入 message，状态 sending
↓
UI 立即展示
↓
请求服务端
↓
服务端返回 server_msg_id / seq / server_time
↓
更新消息状态 success
↓
更新会话摘要
```

#### 验收标准

- 点击发送后消息立即显示。
- 发送中显示 loading。
- 发送成功后状态消失。
- 发送失败后显示失败按钮。
- 点击失败按钮可重发。

### 8.3 图片消息

#### 功能说明

用户可以从相册或相机发送图片。

#### 需求点

- 支持相册选择。
- 支持拍照发送。
- 支持缩略图生成。
- 支持原图发送。
- 支持图片压缩。
- 支持上传进度。
- 支持下载原图。
- 支持大图预览。
- 支持保存到相册。
- 支持转发、收藏、删除、撤回。

#### 图片发送流程

```text
选择图片
↓
生成本地缩略图
↓
保存原图和缩略图到 media/image
↓
插入 message + message_image
↓
UI 立即展示
↓
上传图片到 CDN
↓
获取 cdn_url / media_id
↓
发送消息体到服务端
↓
服务端 ack
↓
更新 send_status / server_msg_id / seq
```

#### 验收标准

- 弱网下图片显示本地缩略图。
- 上传失败后支持重试。
- 已发送图片重启后仍可查看。
- 本地文件丢失时可重新下载。

### 8.4 语音消息

#### 功能说明

用户可以录制并发送语音消息。

#### 需求点

- 支持按住说话。
- 支持上滑取消。
- 支持录音时长限制。
- 支持录音音量动画。
- 支持语音播放。
- 支持播放进度。
- 支持未播放状态。
- 支持听筒 / 扬声器切换，可选。
- 支持语音转文字，可选。

#### 验收标准

- 录音小于最小时长时提示太短。
- 发送成功后可播放。
- 播放完成后状态变为已播放。
- 同一时间只播放一条语音。

### 8.5 视频消息

#### 功能说明

用户可以发送视频消息。

#### 需求点

- 支持相册选择视频。
- 支持拍摄视频。
- 支持视频压缩。
- 支持封面图生成。
- 支持上传进度。
- 支持在线播放或本地播放。
- 支持保存视频。
- 支持转发、删除、撤回。

#### 验收标准

- 视频消息展示封面、时长。
- 上传中展示进度。
- 点击视频可播放。
- 上传失败可重试。

### 8.6 文件消息

#### 功能说明

用户可以发送本地文件。

#### 需求点

- 支持从文件 App 选择文件。
- 展示文件名、扩展名、大小。
- 支持上传进度。
- 支持下载进度。
- 支持本地预览。
- 支持转发、删除、撤回。

#### 验收标准

- 文件大小展示正确。
- 下载完成后可打开。
- 本地文件不存在时可重新下载。

### 8.7 消息撤回

#### 功能说明

用户可以撤回自己发送的消息，管理员可以撤回群成员消息。

#### 需求点

- 支持撤回文本、图片、语音、视频、文件。
- 撤回不物理删除原消息。
- 撤回后展示系统提示。
- 如果撤回的是最后一条消息，需要更新会话摘要。
- 撤回事件需要多端同步。

#### 数据处理

```text
message.revoke_status = 1
插入 message_revoke
更新 conversation.last_message_digest
```

#### UI 展示

```text
你撤回了一条消息
对方撤回了一条消息
管理员撤回了一条成员消息
```

#### 验收标准

- 撤回后原消息不再展示。
- 会话列表摘要更新为撤回提示。
- 对端收到撤回事件后同步更新。
- 重启 App 后撤回状态仍然正确。

### 8.8 消息删除

#### 功能说明

用户可以删除本地聊天消息。

#### 删除类型

| 类型 | 说明 |
|---|---|
| 本地删除 | 只删除当前设备展示 |
| 服务端撤回 | 所有端同步撤回 |
| 清空聊天记录 | 清空某个会话的本地消息 |

#### 数据策略

本地删除建议使用逻辑删除：

```sql
UPDATE message SET is_deleted = 1 WHERE message_id = ?;
```

#### 验收标准

- 删除消息后当前聊天页不再展示。
- 删除不影响对方。
- 清空聊天记录后会话仍可保留。
- 删除最后一条消息后会话摘要正确更新。

### 8.9 消息重发

#### 功能说明

发送失败的消息可以重新发送。

#### 需求点

- 文本消息支持重发。
- 图片 / 视频 / 文件消息支持重新上传或继续上传。
- 重发必须复用 client_msg_id 或建立映射，避免重复消息。
- 重发成功后更新原消息状态。

#### 验收标准

- 断网发送失败后显示失败状态。
- 恢复网络后点击可重发。
- 服务端不会生成重复消息。

### 8.10 消息排序

#### 排序优先级

聊天消息不应只依赖时间戳排序，应按以下顺序：

```text
seq
server_time
local_time
local_id
```

#### 验收标准

- 离线补拉历史消息后顺序正确。
- 多端同步消息顺序正确。
- 弱网情况下不会出现明显乱序。

## 10. 搜索模块需求

### 10.1 全局搜索

#### 功能说明

用户可以搜索联系人、群聊、聊天记录、文件。

#### 搜索范围

- 联系人
- 群聊
- 会话
- 消息文本
- 文件名
- 收藏内容

#### 数据结构

建议单独使用 `search.db`，建立 FTS 表：

```sql
CREATE VIRTUAL TABLE fts_message USING fts5(
    message_id,
    conversation_id,
    sender_name,
    content,
    tokenize = 'unicode61'
);
```

#### 验收标准

- 搜索结果响应快速。
- 搜索索引损坏后可重建。
- 删除消息后搜索结果不再出现。
- 撤回消息不展示原内容。

### 10.2 会话内搜索

#### 功能说明

用户可以在某个聊天会话内搜索消息。

#### 需求点

- 按关键词搜索。
- 按消息类型筛选，可选。
- 按发送人筛选，可选。
- 点击搜索结果跳转到消息位置。

#### 验收标准

- 搜索结果定位准确。
- 跳转历史消息时自动加载上下文。
- 大会话搜索不卡顿。

## 11. 媒体模块需求

### 11.1 媒体文件存储

#### 设计原则

大文件不直接存 SQLite BLOB，数据库只保存元数据，文件实体落盘。

#### 目录结构

```text
media/
  image/
    original/
    thumb/
  video/
    original/
    thumb/
  voice/
  file/
  avatar/
  emoji/
```

#### 命名规则

优先使用：

```text
md5
media_id
url_hash
```

#### 验收标准

- 媒体文件可去重。
- 缓存可清理。
- 数据库记录和文件路径可关联。
- 文件丢失时可重新下载。

### 11.2 媒体上传

#### 需求点

- 支持图片上传。
- 支持视频上传。
- 支持文件上传。
- 支持上传进度。
- 支持失败重试。
- 支持后台继续，可选。

#### 验收标准

- 上传状态准确。
- 上传失败不会导致消息丢失。
- 重启 App 后可以恢复未完成任务。

### 11.3 媒体下载

## 32. 结论

ChatBridge 第一阶段应该重点保证：

1. 会话列表稳定。
2. 消息收发可靠。
3. 本地数据库结构正确。
4. 弱网状态可恢复。
5. 媒体文件不丢失。
6. 撤回 / 删除 / 重发逻辑清晰。
7. 账号数据隔离。
8. 后续可扩展到多端同步和复杂微信级能力。

这个项目的核心不是 UI 仿微信，而是建立一套稳定的 **Apple 平台 IM 本地存储 + 消息同步 + 会话管理 + 媒体管理架构**。
